# Ark Nova Async

An open-source, self-hosted async multiplayer platform for [Ark Nova](https://capstone-games.com/board-games/ark-nova/), inspired by [AsyncTI4](https://asyncti4.com). Play via Discord, asynchronously, over hours or days.

> **Status**: Phase 0 — Foundation. The bot skeleton runs; card database is loaded from community data; full game engine in progress.

---

## Quick Start (Self-Hosted)

### Prerequisites
- Docker + Docker Compose
- A Discord bot application ([create one](https://discord.com/developers/applications))
- The [Next-Ark-Nova-Cards](https://github.com/Ender-Wiggin2019/Next-Ark-Nova-Cards) repo cloned alongside this one

### 1. Clone and configure

```bash
git clone https://github.com/your-org/arknova-async
cd arknova-async

# Clone the community card database (adjacent to this repo)
git clone https://github.com/Ender-Wiggin2019/Next-Ark-Nova-Cards ../Next-Ark-Nova-Cards

# Copy and fill in environment variables
cp .env.example .env
# Edit .env with your Discord token, DB password, JWT secret
```

### 2. Generate the card database

```bash
cd scripts
npm install
npm run convert
cd ..
```

This creates `bot/src/main/resources/cards/base_game.json` with all base game card definitions.

### 3. Start with Docker Compose

```bash
docker compose up --build
```

The bot will connect to Discord, run Flyway migrations, and load the card database automatically.

### 4. Verify

In any Discord server with the bot, run:
```
/arknova-ping
```

## Development Setup

For local development with hot-reload:

```bash
docker compose -f docker-compose.yml -f docker-compose.dev.yml up
```

This exposes:
- Bot API: `http://localhost:8080`
- Frontend: `http://localhost:3000`
- PostgreSQL: `localhost:5432`
- Redis: `localhost:6379`

## Card Database

Card definitions are sourced from the [Next-Ark-Nova-Cards](https://github.com/Ender-Wiggin2019/Next-Ark-Nova-Cards) community repository and converted to JSON by `scripts/src/convert-cards.ts`.

**Card automation progress**: Cards are loaded with human-readable `ability_text` but `effect_code = null`. When a card has no `effect_code`, the bot displays the ability text and asks the player to manually report the effect. Contributors can progressively implement card effects.

### Contributing a Card Effect

1. Pick an unimplemented card from the [issues list](https://github.com/your-org/arknova-async/issues)
2. Add an `effect_code` JSON entry to `base_game.json` following the [Effect DSL docs](docs/effect-dsl.md)
3. Add a test in `CardEffectInterpreterTest.java`
4. Submit a PR
